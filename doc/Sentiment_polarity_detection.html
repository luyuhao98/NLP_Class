4. 网络模型详解

论文[1]中.将CNN运用在nlp问题上,对句子进行语义分类

4.1. 输入层

  • CNN-Static：word2vec直接得到的
  • CNN-non-static： 由word2vec得到后在训练过程中可以根据数据集做调整
  • CNN-rand: 所有word vector都是随机初始化的
  • CNN-multichannel：类似CNN在处理图像问题时的RGB三通道 
      - 可以细粒度的调节非静态通道使它对手头的任务更具体
  - 既不希望Fine tuned的vector距离原始值太远，但同时保留其一定的变化空间。

4.2. 卷基层

  1. 输入层通过卷积操作得到若干个Feature Map
  2. Kernel窗口是一个宽度不变，高度为：3，4，5 的框，只能上下移动
  3. 得到了1个列向量，代表着该卷积核从句子中提取出来的特征

4.5. 分析

　使用non-static版本的word2vec或者GloVe要比单纯的one-hot representation取得的效果好得多； 为了找到最优的过滤器(Filter)大小，可以使用线性搜索的方法。通常过滤器的大小范围在1-10之间，当然对于长句，使用更大的过滤器也是有必要的； Feature Map的数量在100-600之间； 可以尽量多尝试激活函数，实验发现ReLU和tanh两种激活函数表现较佳； 使用简单的1-max pooling就已经足够了，可以没必要设置太复杂的pooling方式； 当发现增加Feature Map的数量使得模型的性能下降时，可以考虑增大正则的力度，如调高dropout的概率； * 为了检验模型的性能水平，多次反复的交叉验证是必要的，这可以确保模型的高性能并不是偶然
